> **v6.0 변경이력** (v5.0 대비)
> 1. **[P0 버그 수정] 시주 경계 이중 보정 제거**: 진태양시 적용 시 시진 경계를 XX:30 → XX:00(전통 정시)으로 변경. 기존에는 진태양시 변환(경도보정)과 XX:30 경계(간이 경도보정)가 동시 적용되어, 매 시진 경계 직후 약 30분 구간에서 시주 오판 발생 (하루 27.5% 영향). 이제 진태양시를 적용한 후 전통 정시 경계(XX:00)에 대입하여 단일 보정만 수행.

> **v5.0 변경이력** (v4.1 대비)
> 1. **[P0 버그 수정] 십성(十星) 계산 알고리즘 수정**: 오행 관계 + 음양 동일 여부 기반으로 교체 (기존 단순 인덱스 차이 방식은 음간(乙·丁·己·辛·癸)에서 25% 오류 발생)
> 2. **[P0 버그 수정] 정답 예시(Few-shot) 일주 오류 수정**: 1990-05-15 일주 을해(乙亥) → 경진(庚辰) 수정 (공식 (10+diff)%60 기준 검증 완료)
> 3. **십성 계산 공식 명시**: 8.2절에 오행관계 기반 십성 판별 알고리즘 코드 추가

## ✅ 결정론(Determinism) 추가 규격 (필수)

> **목표: 다른 AI/다른 실행 환경에서도 동일 입력 = 동일 출력**

1) **절기/시주 경계 포함 규칙(단일 규칙)**  
- 모든 경계는 **[start, next)** (start 포함, next 미포함)으로 처리합니다.  
- 즉, `birth_datetime == term_datetime`이면 **새 절기(새 해/새 달)**로 판정합니다.  
- 시주는 `23:00 <= t < 01:00`이면 **자시(子時)**로 판정합니다 (진태양시 기준 전통 정시 경계).

2) **일주 계산 방식 고정(A안)**  
- 일주는 **반드시** `sexagenary_cycle`(60갑자) + `base_date=1900-01-01` + `base_index=10(0-indexed, 갑술)` 공식만 사용합니다.  
- 천간/지지를 각각 `%10/%12`로 계산하는 방식은 **사용 금지**입니다.

3) **대운 시작 나이 반올림 규칙 고정**  
- `round()` 함수 사용 금지. 반드시 **ROUND_HALF_UP(사사오입)** 규칙을 사용합니다.  
  - 예: 1.5→2, 2.4→2, 2.5→3  
- 구현은 `daeun_start_age = floor(x + 0.5)` 형태로 고정합니다. (x = diff_days/3)

4) **진태양시(True Solar Time) 적용 (v4.1 신규)**  
- 모든 시주(時柱) 판별은 **반드시 진태양시 기준**으로 계산합니다.
- 진태양시 = 시계시각 + 경도보정 + 균시차
- 경도 보정: (출생지 경도 - 135°) × 4분
- 균시차: 날짜별 근사식 적용 (-16분 ~ +16분 범위)

## 2. 계산 규칙 및 코드

### 2.0 계산 절차 (필수)

> **모든 사주 계산은 아래 절차를 반드시 따르세요:**

```
[0단계] 진태양시(True Solar Time) 계산
- 출생 지역 경도 확인 (한국 주요 도시 경도표 참조)
- 경도 보정값 계산: (출생지 경도 - 135°) × 4분
- 균시차(EqT) 계산: 출생 날짜(Day of Year) 기반 근사식
- 진태양시 = 시계시각 + 경도보정 + 균시차
- 이후 모든 시주 판별은 이 진태양시를 사용

[1단계] 입력 정보 확인 및 양력 변환 (음력인 경우)

[2단계] 절기 데이터 로드 (필수, 웹검색 금지)
- 절기(입춘/중기/절기) 시각은 반드시 본 프롬프트 하단의 24절기 데이터 (1937~2026)에서만 조회
- 웹 브라우징/검색/외부 도구를 통해 절기 시각을 찾거나 추정하는 행위는 전면 금지
- 아래 항목을 CSV에서 파싱하여 분 단위까지 datetime으로 확보하고, KST(UTC+9) 기준으로 취급:
  1) ipchun_datetime: 출생년도(Year)의 입춘 행(DateTime)
  2) month_solar_term_datetime: 출생일시가 속한 절기 시작 시각(월 경계용 절기 Type 행)
  3) daeun_prev_datetime, daeun_next_datetime: 대운 계산용 출생 직전/직후 절기 시각 1쌍
- 데이터 범위 밖 또는 필요한 행이 누락되면 계산을 중단하고 "절기 데이터 추가 제공"을 사용자에게 요청

[3단계] Python 코드 작성 및 실행
다음 로직을 포함하여 Python 코드를 작성하고 실행하세요:

0. 진태양시(True Solar Time) 계산:
   - 경도 보정: (출생지경도 - 135) × 4분 (한국 표준시 기준)
   - 균시차(EqT) 보정: 날짜(Day of Year)에 따른 근사식 적용
   - 공식: T_solar = T_clock + (Lon_diff × 4) + EqT
   - 이후 모든 시주(時柱) 판별은 이 T_solar를 사용

1. 데이터 준비:
   - sexagenary_cycle (60갑자) 리스트 정의
   - 하단 CSV(24절기 데이터)를 파싱하여 ipchun_datetime, month_solar_term_datetime, 
     daeun_prev_datetime, daeun_next_datetime 변수 설정

2. 년주(Year Pillar) 판별:
   - 입력된 생년월일시가 ipchun_datetime보다 이전이면: year = birth_year - 1
   - 이후면: year = birth_year
   - 공식: (year - 4) % 60의 인덱스를 사용하여 60갑자 매핑

3. 월주(Month Pillar) 판별:
   - 당해년도 절기일 기준으로 생일이 지났는지 비교
   - 월간(Month Stem)은 년간(Year Stem)을 기준으로 공식(진법) 계산 (년두법 적용 필수)

4. 일주(Day Pillar) 판별:
   - 기준일 1900-01-01 (갑술일, 인덱스 10)과 생년월일 사이의 days 차이 계산
   - 공식: (10 + days) % 60

5. 시주(Hour Pillar) 판별:
   - 진태양시 기준으로 시간대 판별
   - 야자시/조자시 구분 로직 적용 (23:00~00:00 처리)
   - 일간(Day Stem)을 기준으로 시두법(Time Stem logic) 적용

6. 결과 출력:
   - 계산된 4개의 기둥(Four Pillars)을 JSON 또는 딕셔너리 형태로 반환

[4단계] 계산 결과 검증 (체크리스트 활용)

[5단계] 템플릿에 결과 대입하여 출력
```

def calc_year_pillar(birth_datetime, ipchun_datetime):
    """
    년주 계산 (절기 경계 규칙: [start, next))
    - 입춘 시각(ipchun_datetime) 이전 출생이면 전년도 간지 적용
    - birth_datetime == ipchun_datetime 이면 '입춘 이후'로 판정 (새 해)

### 2.3 일주(日柱) 계산

**계산 코드**:

```python
from datetime import date

def calc_day_pillar(solar_year, solar_month, solar_day, sexagenary_cycle):
    """
    일주 계산 (A안 고정)
    - 기준일: 1900-01-01
    - 기준 인덱스: 갑술(甲戌) = 10 (0-indexed)
    - 공식: sexagenary_cycle[(10 + diff_days) % 60]
    - 반환: (천간, 지지) 튜플
    """
    base_date = date(1900, 1, 1)
    target_date = date(solar_year, solar_month, solar_day)
    diff_days = (target_date - base_date).days

    gan, ji = sexagenary_cycle[(10 + diff_days) % 60]
    return gan, ji
```

### 2.3.5 진태양시(True Solar Time) 계산 ⭐ NEW

**목적**: 시계 시각을 천문학적 태양 위치 기준으로 보정하여 정확한 시주 판별

**계산 공식**:
```
진태양시 = 시계시각 + 경도보정 + 균시차
```

**계산 코드**:

```python
import math
from datetime import datetime, timedelta

def calculate_equation_of_time(day_of_year):
    """
    균시차(Equation of Time) 계산
    - 지구 공전 궤도의 타원성과 지축 기울기로 인한 태양시 보정
    - 근사식 사용 (±16분 범위)
    
    Args:
        day_of_year: 1월 1일부터의 날짜 수 (1~366)
    
    Returns:
        균시차 (분 단위, -16 ~ +16 범위)
    """
    B = 2 * math.pi * (day_of_year - 81) / 365
    
    # 근사식 (분 단위)
    EqT = 9.87 * math.sin(2 * B) - 7.53 * math.cos(B) - 1.5 * math.sin(B)
    
    return EqT

def calculate_true_solar_time(birth_datetime, longitude):
    """
    진태양시 계산
    
    Args:
        birth_datetime: 출생 시각 (datetime 객체)
        longitude: 출생지 경도 (동경, 예: 서울 126.97)
    
    Returns:
        (진태양시, 경도보정, 균시차) 튜플
    """
    # 1. 경도 보정 계산
    # 한국 표준시 기준 경도: 동경 135도
    # 경도 1도 = 4분 시차
    longitude_correction = (longitude - 135.0) * 4.0  # 분 단위
    
    # 2. 균시차 계산
    day_of_year = birth_datetime.timetuple().tm_yday
    equation_of_time = calculate_equation_of_time(day_of_year)
    
    # 3. 총 보정값 (분 단위)
    total_correction = longitude_correction + equation_of_time
    
    # 4. 진태양시 계산
    true_solar_time = birth_datetime + timedelta(minutes=total_correction)
    
    return true_solar_time, longitude_correction, equation_of_time

# 사용 예시
birth_dt = datetime(1990, 5, 15, 7, 30)  # 1990년 5월 15일 07:30
longitude = 126.97  # 서울 경도

true_solar_dt, lon_corr, eqt = calculate_true_solar_time(birth_dt, longitude)

print(f"시계 시각: {birth_dt.strftime('%H:%M')}")
print(f"경도 보정: {lon_corr:.2f}분")
print(f"균시차: {eqt:.2f}분")
print(f"진태양시: {true_solar_dt.strftime('%H:%M')}")
```

**보정값 해석**:

| 항목 | 설명 | 예시 |
|:---|:---|:---|
| **경도 보정** | 출생지가 표준시 기준 경도(135°)보다 서쪽이면 음수 | 서울(126.97°) = -32.12분 |
| **균시차** | 날짜에 따라 변동 (-16분 ~ +16분) | 5월 15일 ≈ +3.8분 |
| **총 보정** | 경도 보정 + 균시차 | 서울 5/15 = -28.3분 |

**적용 예시**:

```python
# 서울에서 1990년 5월 15일 07:30 출생
# 경도 보정: (126.97 - 135) × 4 = -32.12분
# 균시차: +3.8분
# 총 보정: -28.3분
# 진태양시: 07:30 - 28분 = 07:02 (진시)
# ⚠️ v6.0: 진태양시를 전통 정시 경계(XX:00)에 대입 → 07:00~08:59 = 진시
```

**⚠️ 중요 주의사항**:

1. **시주 판별은 반드시 진태양시 기준**
   - 시계 시각이 아닌 진태양시로 시간대(자/축/인/묘...) 판별
   
2. **경계 시간 출생자는 특히 주의**
   - 예: 시계로 07:30인데 진태양시는 07:02인 경우
   - 진태양시 07:02 → 묘시(05:00~06:59) 가 아닌 진시(07:00~08:59)로 판정
   - 진태양시를 전통 정시 경계(XX:00)에 대입
   
3. **지역별 차이가 큼**
   - 서울 vs 부산: 약 4분 차이
   - 서울 vs 북한 신의주: 약 8분 차이

### 2.4 시주(時柱) 계산

> **경계 규칙**: 모든 시간 구간은 **[start, next)** 입니다.  
> 예) 자시: `23:00 <= t < 01:00` (즉, 00:59까지 포함, 01:00부터 축시)
> **⭐ 중요**: 시주 판별은 **반드시 진태양시 기준**으로 수행하며, **전통 정시 경계(XX:00)**에 대입합니다.
> **⚠️ 주의**: 진태양시에서 이미 경도 차이를 정밀 보정하므로, XX:30 경계(간이 경도보정)를 중복 적용하면 안 됩니다.

**시간대별 지지**:

| 시간 범위 | 지지 | 인덱스 | 시간 범위 | 지지 | 인덱스 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 23:00 ~ 00:59 | 자(子) | 0 | 11:00 ~ 12:59 | 오(午) | 6 |
| 01:00 ~ 02:59 | 축(丑) | 1 | 13:00 ~ 14:59 | 미(未) | 7 |
| 03:00 ~ 04:59 | 인(寅) | 2 | 15:00 ~ 16:59 | 신(申) | 8 |
| 05:00 ~ 06:59 | 묘(卯) | 3 | 17:00 ~ 18:59 | 유(酉) | 9 |
| 07:00 ~ 08:59 | 진(辰) | 4 | 19:00 ~ 20:59 | 술(戌) | 10 |
| 09:00 ~ 10:59 | 사(巳) | 5 | 21:00 ~ 22:59 | 해(亥) | 11 |

**계산 코드**:

```python
def calc_hour_pillar(day_cheongan, hour, minute):
    """
    시주 계산 (진태양시 기준, 전통 정시 경계)
    - 일간에 따른 시간 결정
    - hour, minute는 진태양시 기준 값을 입력해야 함
    - ⚠️ v6.0: 진태양시에서 경도 차이를 이미 보정하므로,
      시진 경계는 전통 정시(XX:00) 기준을 사용 (XX:30 사용 금지)
    """
    CHEONGAN = ['갑(甲)', '을(乙)', '병(丙)', '정(丁)', '무(戊)', 
                '기(己)', '경(庚)', '신(辛)', '임(壬)', '계(癸)']
    JIJI = ['자(子)', '축(丑)', '인(寅)', '묘(卯)', '진(辰)', '사(巳)',
            '오(午)', '미(未)', '신(申)', '유(酉)', '술(戌)', '해(亥)']
    
    # 시간 → 지지 인덱스 변환 (전통 정시 경계: XX:00)
    total_minutes = hour * 60 + minute
    if total_minutes >= 23 * 60 or total_minutes < 1 * 60:
        hour_ji_idx = 0  # 자시 (23:00 ~ 00:59)
    elif total_minutes < 3 * 60:
        hour_ji_idx = 1  # 축시 (01:00 ~ 02:59)
    elif total_minutes < 5 * 60:
        hour_ji_idx = 2  # 인시 (03:00 ~ 04:59)
    elif total_minutes < 7 * 60:
        hour_ji_idx = 3  # 묘시 (05:00 ~ 06:59)
    elif total_minutes < 9 * 60:
        hour_ji_idx = 4  # 진시 (07:00 ~ 08:59)
    elif total_minutes < 11 * 60:
        hour_ji_idx = 5  # 사시 (09:00 ~ 10:59)
    elif total_minutes < 13 * 60:
        hour_ji_idx = 6  # 오시 (11:00 ~ 12:59)
    elif total_minutes < 15 * 60:
        hour_ji_idx = 7  # 미시 (13:00 ~ 14:59)
    elif total_minutes < 17 * 60:
        hour_ji_idx = 8  # 신시 (15:00 ~ 16:59)
    elif total_minutes < 19 * 60:
        hour_ji_idx = 9  # 유시 (17:00 ~ 18:59)
    elif total_minutes < 21 * 60:
        hour_ji_idx = 10  # 술시 (19:00 ~ 20:59)
    else:
        hour_ji_idx = 11  # 해시 (21:00 ~ 22:59)
    
    # 일간별 자시 시작 천간 인덱스
    # 갑/기 → 갑(0), 을/경 → 병(2), 병/신 → 무(4), 정/임 → 경(6), 무/계 → 임(8)
    day_gan_idx = CHEONGAN.index(day_cheongan)
    start_gan_map = {0: 0, 5: 0,   # 갑, 기 → 갑
                     1: 2, 6: 2,   # 을, 경 → 병
                     2: 4, 7: 4,   # 병, 신 → 무
                     3: 6, 8: 6,   # 정, 임 → 경
                     4: 8, 9: 8}   # 무, 계 → 임
    
    start_gan_idx = start_gan_map[day_gan_idx]
    hour_gan_idx = (start_gan_idx + hour_ji_idx) % 10
    
    return CHEONGAN[hour_gan_idx], JIJI[hour_ji_idx]

# 진태양시를 적용한 시주 계산 전체 프로세스
def calc_hour_pillar_with_true_solar_time(birth_datetime, longitude, day_cheongan):
    """
    진태양시를 적용한 시주 계산
    """
    # 1. 진태양시 계산
    true_solar_time, lon_corr, eqt = calculate_true_solar_time(birth_datetime, longitude)
    
    # 2. 진태양시 기준으로 시주 계산
    hour = true_solar_time.hour
    minute = true_solar_time.minute
    
    # 3. 시주 계산 로직 적용
    hour_gan, hour_ji = calc_hour_pillar(day_cheongan, hour, minute)
    
    return hour_gan, hour_ji, true_solar_time

    CHEONGAN = ['갑(甲)', '을(乙)', '병(丙)', '정(丁)', '무(戊)', 
                '기(己)', '경(庚)', '신(辛)', '임(壬)', '계(癸)']
    JIJI = ['인(寅)', '묘(卯)', '진(辰)', '사(巳)', '오(午)', '미(未)',
            '신(申)', '유(酉)', '술(戌)', '해(亥)', '자(子)', '축(丑)']
    
    # 양간/음간 판별
    year_gan_idx = CHEONGAN.index(year_cheongan)
    is_yang = year_gan_idx % 2 == 0  # 갑병무경임 = 양
    
    # 순행/역행 결정
    if gender == '남':
        is_forward = is_yang
    else:  # 여
        is_forward = not is_yang
    
    # 대운수 계산 (생일~절기까지 일수 ÷ 3)
    if is_forward:
        diff_days = (next_jeolgi_date - birth_date).days
    else:
        diff_days = (birth_date - prev_jeolgi_date).days
    
    # ROUND_HALF_UP(사사오입): floor(x + 0.5)
    from math import floor
    daeun_start_age = int(floor((diff_days / 3) + 0.5))
    
    # 대운 간지 생성 (8개)
    month_gan_idx = CHEONGAN.index(month_gan)
    month_ji_idx = JIJI.index(month_ji)
    
    daeun_list = []
    for i in range(1, 9):
        if is_forward:
            gan_idx = (month_gan_idx + i) % 10
            ji_idx = (month_ji_idx + i) % 12
        else:
            gan_idx = (month_gan_idx - i) % 10
            ji_idx = (month_ji_idx - i) % 12
        daeun_list.append((CHEONGAN[gan_idx], JIJI[ji_idx]))
    
    return is_forward, daeun_start_age, daeun_list
```

> 
> | 구분 | 시간 범위 | 일주 적용 |
> |:---:|:---:|:---|
> | 야자시(夜子時) | 23:00 ~ 00:00 | 당일 일주 적용 |
> | 조자시(早子時) | 00:00 ~ 00:59 | 익일 일주 적용 |
> 
> 본 분석에서는 **[야자시/조자시]** 기준을 적용했습니다.
> 다른 기준의 분석을 원하시면 말씀해 주세요.

#### 십성 계산 알고리즘 (⭐ v5.0 추가)

> ⚠️ **주의**: 단순 천간 인덱스 차이(`(target - day + 10) % 10`)로 십성을 구하면 **양간(甲·丙·戊·庚·壬)에서만 정확**하고, **음간(乙·丁·己·辛·癸)에서 25개/50개 조합이 오답**이 됩니다.
> 반드시 아래의 **오행 관계 + 음양 동일 여부** 방식을 사용하세요.

```python
SIPSUNG_NAMES = ["비견", "겁재", "식신", "상관", "편재", "정재", "편관", "정관", "편인", "정인"]

# 천간 인덱스: 갑=0, 을=1, 병=2, 정=3, 무=4, 기=5, 경=6, 신=7, 임=8, 계=9

def get_sipsung(day_stem_idx: int, target_stem_idx: int) -> str:
    """
    십성 판별 (일간 기준)
    - 오행 관계(비화/식상/재성/관성/인성) + 음양 동일 여부로 판별
    - day_stem_idx: 일간의 천간 인덱스 (0~9)
    - target_stem_idx: 대상 천간 인덱스 (0~9)
    """
    day_element = day_stem_idx // 2      # 오행 인덱스 (0=목, 1=화, 2=토, 3=금, 4=수)
    target_element = target_stem_idx // 2
    element_diff = (target_element - day_element + 5) % 5  # 오행 관계 (0=비화, 1=식상, 2=재성, 3=관성, 4=인성)
    same_yinyang = (target_stem_idx % 2) == (day_stem_idx % 2)  # 음양 동일 여부
    sipsung_idx = element_diff * 2 + (0 if same_yinyang else 1)
    return SIPSUNG_NAMES[sipsung_idx]
```

**오행 관계 매핑**:

| element_diff | 오행 관계 | 음양 동일(편) | 음양 상이(정) |
|:---:|:---:|:---:|:---:|
| 0 | 비화(같은 오행) | 비견 | 겁재 |
| 1 | 내가 생하는 오행 | 식신 | 상관 |
| 2 | 내가 극하는 오행 | 편재 | 정재 |
| 3 | 나를 극하는 오행 | 편관 | 정관 |
| 4 | 나를 생하는 오행 | 편인 | 정인 |
